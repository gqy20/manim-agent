# Scene Patterns

## Quick Demo

- Show one object or relationship.
- Use 1 to 3 short beats.
- Avoid narration longer than the animation itself.

## Concept Explainer

- Start with the object or question.
- Introduce one relationship at a time.
- End with a short takeaway frame.

## Proof Walkthrough

- Show givens first.
- Reveal each transformation in order.
- Highlight only one algebraic or geometric change per beat.
- End with the final statement clearly isolated.

## Function Visualization

- Build axes first if they matter.
- Introduce the base graph, then parameter changes.
- Keep labels close to the graph element they describe.

## Geometry Construction

- Draw base shapes before annotations.
- Add right-angle marks, equal marks, and labels after the geometry exists.
- Use movement or color to show correspondence, not extra paragraphs.
